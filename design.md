# ランチ＆ディナーレコメンドアプリ デザイン計画

## アプリコンセプト
毎日のお昼ご飯を記録し、栄養バランスや食事の傾向を分析して、夜ご飯のおすすめメニューを提案するアプリ。

---

## スクリーン一覧

### 1. ホーム画面 (Home)
- 今日のランチ記録サマリー
- 今日のおすすめディナー（クイックアクセス）
- 最近の食事履歴（直近3日間）

### 2. ランチ記録画面 (Record Lunch)
- 料理名入力フィールド
- カテゴリ選択（和食、洋食、中華、その他）
- メモ欄（オプション）
- 記録ボタン

### 3. おすすめ画面 (Dinner Recommendation)
- 今日のランチに基づいたおすすめディナー
- おすすめ理由の表示
- 複数の候補から選択可能
- 「これにする」ボタンで夜ご飯を記録

### 4. 履歴画面 (History)
- カレンダー形式で過去の食事を表示
- 日付タップで詳細表示
- ランチ・ディナーの両方を表示

---

## 主要コンテンツと機能

### ホーム画面
- **今日のランチカード**: 記録済みの場合は料理名とカテゴリを表示、未記録の場合は「ランチを記録」ボタン
- **おすすめディナーカード**: ランチ記録後に表示、タップで詳細画面へ
- **クイック履歴リスト**: 直近の食事記録をリスト表示

### ランチ記録画面
- **料理名入力**: テキストフィールド（必須）
- **カテゴリ選択**: セグメントコントロール（和食/洋食/中華/その他）
- **メモ**: テキストエリア（オプション）
- **記録ボタン**: プライマリアクションボタン

### おすすめ画面
- **おすすめカード**: 3つのおすすめメニューをカード形式で表示
- **理由表示**: 「お昼が〇〇だったので、△△がおすすめです」
- **選択ボタン**: 各カードに「これにする」ボタン

### 履歴画面
- **カレンダービュー**: 月表示、食事記録のある日にドット表示
- **日別詳細**: 選択日のランチ・ディナーを表示

---

## 主要ユーザーフロー

### フロー1: ランチを記録する
1. ホーム画面で「ランチを記録」をタップ
2. 記録画面で料理名を入力
3. カテゴリを選択
4. 「記録する」をタップ
5. ホーム画面に戻り、おすすめディナーが表示される

### フロー2: おすすめディナーを確認・選択する
1. ホーム画面でおすすめカードをタップ
2. おすすめ画面で3つの候補を確認
3. 気に入ったメニューの「これにする」をタップ
4. 夜ご飯として記録される
5. ホーム画面に戻る

### フロー3: 過去の食事を確認する
1. 履歴タブをタップ
2. カレンダーで日付を選択
3. その日のランチ・ディナーを確認

## データモデル

### MealRecord
- id: string
- date: string (YYYY-MM-DD)
- mealType: lunch or dinner
- dishName: string
- category: japanese, western, chinese, other
- note: string (optional)
- createdAt: timestamp

### DinnerRecommendation
- id: string
- lunchId: string (reference)
- recommendations: Array of name and reason
- selectedId: string (optional)
- createdAt: timestamp
